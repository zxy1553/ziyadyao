## 🔧 Part 1: Maintainer 自动化维护

### 问题
- 代码迭代 → API 变化 → 知识库过时 → Agent 回答不准确

### 解决方案：配置路径 → 自动抽取 → 按模块生成

**Maintainer 工作流程图**：
```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   📁 配置文件    │     │   🔍 自动抽取    │     │   📚 知识库      │
│                 │     │                 │     │                 │
│  watch_paths:   │ ──► │  扫描 .cpp/.lua │ ──► │  Core/API/      │
│  - /Core/Status │     │  AST 解析结构   │     │  Effect/API/    │
│  - /Core/Buff   │     │  按模块分类     │     │  impl.json      │
│  - /Targeting   │     │                 │     │                 │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### 核心能力
| 功能 | 说明 |
|------|------|
| **路径监控** | 配置关注的代码目录 |
| **自动抽取** | 扫描 cpp/lua，解析 API 结构 |
| **模块分类** | 按功能域自动归类 |
| **索引更新** | 自动生成/更新 impl.json |

## 🚀 Part 2: 一键配置推广

### 场景：其他团队（3C、AI 等）想接入

### 传统 vs 一键配置

| 对比维度 | 传统方式 | 一键配置 |
|----------|----------|----------|
| **知识库搭建** | 手动整理，耗时数周 | 配置路径，自动生成 |
| **结构设计** | 从零设计 | 复用现有模板 |
| **后续维护** | 手动同步 | Maintainer 自动维护 |

### 接入 4 步流程

```
Step 1                 Step 2                 Step 3                 Step 4
┌──────────┐          ┌──────────┐          ┌──────────┐          ┌──────────┐
│ 配置路径  │    ──►   │ 运行工具  │    ──►   │ 知识库    │    ──►   │ 配置Agent │
│          │          │          │          │ 自动生成  │          │          │
│ config:  │          │ python   │          │ 3C-KB/   │          │ 复用模板  │
│ /3C/Char │          │ maintain │          │ ├─Char/  │          │ 改路径   │
│ /3C/Cam  │          │ er.py    │          │ ├─Cam/   │          │ 即可用   │
└──────────┘          └──────────┘          └──────────┘          └──────────┘
```

## 🔄 Part 3: 持续迭代闭环

```
        使用 Agent
            │
            ▼
    ┌───────────────┐
    │  发现问题     │ ◄──────────────────────┐
    │  API 缺失     │                        │
    └───────┬───────┘                        │
            │                                │
            ▼                                │
    ┌───────────────┐      ┌───────────────┐ │
    │  反馈记录     │ ──►  │  Maintainer   │ │
    │  哪些需补充   │      │  更新知识库   │ │
    └───────────────┘      └───────┬───────┘ │
                                   │         │
                                   ▼         │
                           ┌───────────────┐ │
                           │  知识库升级   │ │
                           │  Agent 更准   │─┘
                           └───────────────┘
```

**核心理念**：越用越准，持续进化

---

## 📊 推广价值总结

| 维度 | 现状（战斗模块） | 推广后 |
|------|-----------------|--------|
| **覆盖范围** | 战斗系统 | 3C、AI、UI 等多模块 |
| **接入成本** | 从零搭建 | 配置 + 一键生成 |
| **维护方式** | 半自动 | 全自动 |
| **SubAgent** | 战斗 Dev/Debug | 每模块可复用模板 |
