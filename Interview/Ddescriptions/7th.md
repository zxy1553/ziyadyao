### 三栏并列展示

| 方案 | TAPD AI (网页) | Default Agent | Debug Agent |
|------|---------------|---------------|-------------|
| **图标** | 🔴 | 🟡 | 🟢 |

## 💡 关键差异总结（底部横幅）

| Default Agent 的问题 | Debug Agent 的解决 |
|---------------------|-------------------|
| 不知道分析谁的日志 | 按 BattleID 自动过滤 |
| 分析无关时间段 | 按时间戳智能切分 |
| 没有固定分析流程 | Debug 总纲规范化 |
| 不给关键日志引用 | 输出文档 + 行号 |

## 🗣️ 演讲稿（50秒）

> **三方对比（20秒）**
> "Debug 我们对比三个方案：TAPD AI 只能读 Bug 描述，给'检查网络配置'这种通用建议。原生 Agent 能读日志，但没方向——经常分析错误的角色，看了半天最后告诉你'建议优化代码'。Debug Agent 不一样，它有完整工作流。"

> **工作流展示（25秒）**
> "第一步给 BattleID，自动从智研拉日志。第二步日志太大读不了，自动切分到 10MB 以内。第三步按 Debug 总纲的固定流程：先确认角色，再定位时间段，再找关键事件。最后输出分析文档，带关键日志行号。"

> **总结（5秒）**
> "这就是**自动化 + 规范化**带来的差异。"
